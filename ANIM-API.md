# ANIM-API v3 — документация для написания сюжетных canvas-анимаций

Проект: сайт-плеер альбома «Cupsize — заставь меня плакать». 26 треков, у каждого —
сюжетная анимация на canvas, рассказывающая историю песни. Эстетика: карандашный
скетч на старой бумаге. Чернила `#1a1612`, красный `#c8281a`, бумага `#f2ede6`.
Линии «дышат» (boil ~12fps), движение — плавное 60fps. Анимация синхронизирована
с реальным временем трека и реагирует на музыку (бит/бас).

## Окружение (глобальные, уже определены — НЕ переопределять)

Из main.js: `ctx`, `canvas`, `cW()`, `cH()` (ширина/высота в px), `PAPER`, `INK`, `RED`,
`rnd(a,b)` (случайное число), `grain()` (зерно плёнки), `animRunning` (bool — флаг
работы анимации), `rafId`, `currentTrack`, `howl`, `kPrevIdx`, `karaokeLines`.

Из ядра v3 (файл `story-core.js` — прочитай его, чтобы знать точную семантику):

### Тайминг
- `prog()` → 0..1 — плавный прогресс трека (по howl.seek/duration, лерп-сглаженный). Вызывать РАЗ за кадр.
- `_pg = -1` — сброс лерпа; делать ПЕРВОЙ строкой своей функции.
- `T()` / локальный счётчик `t` — кадры для непрерывного движения.
- `EZ.io(t) EZ.out(t) EZ.in(t) EZ.bounce(t) EZ.pulse(t)` — изинги.

### Аудио-реактивность (ОБЯЗАТЕЛЬНО использовать минимум в 2 сценах)
- `AUD.beat` — 0..1, вспыхивает на ударе баса, затухает. Пульс, тряска, вспышки.
- `AUD.bass / AUD.mid / AUD.high / AUD.level` — 0..1, сглаженные уровни.
- `kick(0.08)` → множитель масштаба `1 + AUD.beat*0.08` — для пульсации объектов.

### Перо и цвет
- `ink(a, w)` / `red(a, w)` / `acc(a, w)` — strokeStyle (+lineWidth); acc = акцентный цвет трека.
- `fink(a)` / `fred(a)` / `facc(a)` — fillStyle.
- `txt(s, x, y, size, a, col?)` — рукописный текст (Caveat), по центру.
- `caption(s, W, H, a?)` — подпись-цитата внизу кадра (1 на сцену, ≤34 символов).
- `writeOn(s, x, y, size, lp, a, col?)` — текст «пишется» по мере lp (эффект почерка).
- `flash(a, col?)` — вспышка на весь экран (умеренно!).

### Скетч-примитивы (стабильный дребезг, безопасно вызывать каждый кадр)
- `sl(x1,y1,x2,y2,w?)` — линия; `scir(cx,cy,r,w?)` — круг; `sarc(cx,cy,r,a0,a1,w?)` — дуга;
- `sellipse(cx,cy,rx,ry,w?)` — эллипс; `spath([[x,y],...],close?,w?)` — ломаная; `srect(x,y,w,h,w?)`.
- `ww(amp)` — «дребезг» числа (детерминирован внутри кадра). Для своих мелких смещений.

### Сцены (раскадровка)
- `act(p, from, to, (lp, a) => {...})` — сцена в интервале прогресса [from..to] с
  автокроссфейдом по краям. lp = 0..1 локальный прогресс сцены, a = альфа.
  Интервалы берёшь из раскадровки. Соседние сцены могут перекрываться на 0.01-0.03.
- `scenes(p, [fn, fn, ...])` — равные доли (если интервалы не важны).
- `camShake(amp)` — тряска камеры (внутри ctx.save()/restore()!).

### Персонажи
- `figure(x, y, opts)` — человечек ростом ~100px при s=1 (голова на y-39..-21, ноги до y+50). opts:
  - `s` масштаб, `a` альфа, `lean` наклон (рад), `flip` отзеркалить, `headTilt` наклон головы
  - `ph` фаза ходьбы (синус), `step` амплитуда шага, `swing` мах руками
  - `raise` 0..1 поднять обе руки, `raiseL`/`raiseR` по отдельности
  - `sit: true` — сидит; `fem: true` — девушка (платье+волосы), `hairFlow` развевание, `hairLong` длина
  - `face` — эмоция: `'happy' 'sad' 'angry' 'cry' 'flat' 'x'(мёртв) 'shock' 'love' 'calm'`
  - `hair:'none'` — без волос
- `drawFace(x, y, type, a)` — лицо отдельно (на крупных планах рисуй голову сам + drawFace).

### Частицы
```js
const parts = mkParts(200);              // в инициализации
parts.spawn({x,y,vx,vy,g,life,r,col:[r,g,b],type,rot,vr,drag,txt}); // в кадре по событию
parts.step(); parts.draw();              // раз за кадр
```
types: `dot line star heart note smoke shard petal drop`. (note = ♪, txt можно заменить)

### Погода (создать в init, вызывать в кадре)
- `const rain = mkRain(40);` → `rain(W, H, alpha, wind)`
- `const snow = mkSnow(60);` → `snow(W, H, alpha)`

### Реквизит (готовые скетч-объекты)
- `cloud(cx,cy,r,a)` облако; `bldg(x,yTop,w,base,a,rows)` дом с горящими окнами;
- `windowFrame(x,y,w,h,a)` окно; `balloon(x,y,r,ax,ay,a)` шарик на нитке к точке (ax,ay);
- `star5(x,y,r,a,fill?)` звезда; `drawHeart(x,y,r,a,col?)` сердце;
- `drawBike(x,y,s,a,rot)` велосипед (rot — вращение колёс); `drawSwing(x,y,h,ang,a)` качели (ang — качание);
- `drawTree(x,gy,s,a,wind)` дерево; `drawBottle(x,y,s,a,tilt)` бутылка; `drawKnife(x,y,s,ang,a)` нож;
- `drawFlower(x,y,s,a,sway)` цветок; `drawCat(x,y,s,a,tail)` кот; `drawDog(x,y,s,a,bark)` собака;
- `drawCross(x,y,s,a)` крест; `drawCandle(x,y,s,a,flicker)` свеча; `drawPhone(x,y,s,a,glow)` телефон;
- `drawHouse(x,gy,s,a)` домик; `drawMoon(x,y,r,a)`; `drawSun(x,y,r,a)`.
- `dust(W,H,a)` пылинки; `ground(W,H,gy,a)` линия земли; `vignettePulse(W,H,base)` виньетка.

## Каркас функции (СТРОГО эта структура)

```js
function NAME(){
  let t=0; _pg=-1;
  const parts=mkParts(200);            // если нужны частицы
  const snow=mkSnow(50);               // если нужна погода
  // ...вся инициализация со случайностями ТОЛЬКО здесь (rnd можно)
  function f(){
    if(!animRunning)return;
    const W=cW(),H=cH(); bg(W,H); const p=prog(); t++;
    // постоянные элементы (фон/погода) — если есть во всех сценах
    act(p,0.00,0.10,(lp,a)=>{ /* сцена 1 */ });
    act(p,0.09,0.35,(lp,a)=>{ /* сцена 2 */ });
    // ... последняя сцена ДО 1.0
    vignettePulse(W,H,0.18); grain();
    rafId=requestAnimationFrame(f);
  }
  f();
}
```

## Заповеди

1. Ровно ОДНО объявление верхнего уровня: `function NAME(){...}`. Никаких глобальных
   констант/функций снаружи. Вспомогательные функции — внутри NAME.
2. `Math.random()` / `rnd()` — ТОЛЬКО в инициализации, НИКОГДА внутри `f()`
   (иначе стробоскоп). Дрожание линий уже встроено в sl/scir/sarc через ww().
3. Все координаты — доли W/H (`W*0.5`, `H*0.3`). Канвас от 320×200 до 1600×900,
   пропорции непредсказуемы. Размеры объектов тоже масштабируй от Math.min(W,H) при
   необходимости.
4. Движение НЕПРЕРЫВНОЕ: позиции от `t` (постоянные циклы) и от `lp` через EZ-изинги
   (сюжетные перемещения). Никаких телепортаций.
5. Аудио: AUD.beat/bass/kick() минимум в 2 сценах (тряска, пульс сердца, вспышка,
   подпрыгивание, дрожь предметов).
6. Лица персонажей МЕНЯЮТСЯ по сюжету (happy → shock → cry...). Детали фона
   обязательны: окна, дым, птицы, мусор, звёзды — сцена не должна быть пустой.
   Но ≤ ~600 примитивов на кадр (производительность).
7. Подписи: cyrillic-цитаты из песни через caption(), макс 1 на сцену. Допустим
   крупный «титульный» текст через txt()/writeOn() в кульминации.
8. Чёрный юмор и трагизм песни передавать образами (силуэты, символы, кресты, нимбы),
   без натурализма: это карандашный скетч.
9. Запрещено: `setTimeout/setInterval`, `Date.now`, `performance.now`, `new Image`,
   `fetch`, `document.*`, бесконечные `while`. Только canvas-рисование через ctx и API выше.
10. Финал кадра ВСЕГДА: `vignettePulse(W,H,0.18); grain(); rafId=requestAnimationFrame(f);`
    и вызов `f();` в конце NAME.
11. Объём: 90–230 строк. Любая сцена внутри себя должна эволюционировать от lp=0 к lp=1
    (не статичная картинка!).
12. ctx.save()/ctx.restore() — парные. globalAlpha не трогать напрямую (act() сам управляет).

## Чек-лист ревьюера

1. Ровно одна верхнеуровневая конструкция: `function NAME(){...}`; имя совпадает с заданием.
2. Каркас: `_pg=-1` в начале; `if(!animRunning)return;` первой строкой f();
   `bg(W,H)`; `const p=prog()` один раз; хвост кадра `vignettePulse → grain → rafId=requestAnimationFrame(f)`; `f()` вызвана.
3. Нет rnd()/Math.random() внутри f() и вложенных в f() функций.
4. Нет запрещённых API (setTimeout, Date.now, document, fetch, while(true)...).
5. Все вызовы хелперов существуют в ANIM-API (проверь имена и сигнатуры!).
   Частые ошибки: drawHeart (не heart!), sellipse, mkParts().spawn — поля объектом.
6. Координаты относительные (нет «голых» констант больше ~60 для позиций).
7. AUD.beat/bass/kick используется ≥2 раз.
8. act()-интервалы: первая сцена с 0.0, последняя до 1.0, разрывов нет (объединение
   интервалов покрывает [0,1]).
9. Баланс скобок/кавычек, валидный ES2020. Шаблонные строки с ${} корректны.
10. ctx.save/restore парные. Внутри forEach/for нет утечки состояния пера.
11. Подписи на русском, без опечаток, осмысленные цитаты песни.
12. Производительность: нет циклов >800 итераций на кадр, частиц ≤ ~300.
